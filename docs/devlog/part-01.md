# 백엔드 개발자가 바이브 코딩으로 프론트 개발을 — Part 1

## 시작 배경
- API 문서, 테스트 결과가 아닌 실제 유저가 쓸 화면을 만드는 것을 목표
- 바이브 코딩

## 기술 스택
바이브 코딩의 특성상, 스택 선택의 기준은 단 하나였다 — **AI가 잘 아는 것**.
직접 비교 분석하는 대신 Claude에게 "백엔드 API와 연동하는 웹 프론트엔드를 만들고 싶다"고 했고, 아래 스택으로 프로젝트가 세팅됐다.

### 런타임 / 언어

| 기술 | 버전 | 역할 |
|------|------|------|
| **TypeScript** | ~5.9.3 | 메인 언어. JavaScript에 정적 타입을 추가한 것으로, 백엔드의 Kotlin과 비슷하게 컴파일 타임에 타입 오류를 잡아줌 |

---

### UI

| 기술 | 버전 | 역할 |
|------|------|------|
| **React** | ^19.2.4 | UI 프레임워크. 화면을 컴포넌트 단위로 쪼개서 관리 |
| **react-dom** | ^19.2.4 | React를 실제 브라우저 DOM에 렌더링하는 패키지 |
| **CSS Modules** | — | 컴포넌트별로 스타일을 격리. 클래스명 충돌 없음 |

---

### 라우팅

| 기술 | 버전 | 역할 |
|------|------|------|
| **React Router DOM** | ^7.13.2 | URL 경로에 따라 다른 페이지 컴포넌트를 렌더링. Spring의 `@RequestMapping`과 유사한 역할 |

---

### HTTP 통신

| 기술 | 버전 | 역할 |
|------|------|------|
| **Axios** | ^1.13.6 | 백엔드 API를 호출하는 HTTP 클라이언트. Java의 `RestTemplate` / `WebClient`와 같은 역할 |
